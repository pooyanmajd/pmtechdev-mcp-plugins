# Support

Use the repository's [Discussions](https://github.com/pooyanmajd/pmtechdev-mcp-plugins/discussions) for installation help, usage questions, and design conversations.

Open a [bug report](https://github.com/pooyanmajd/pmtechdev-mcp-plugins/issues/new?template=bug.yml) only for a reproducible defect that can be demonstrated with synthetic data. Use the [feature proposal](https://github.com/pooyanmajd/pmtechdev-mcp-plugins/issues/new?template=feature.yml) for focused improvements or new plugin ideas.

Report suspected vulnerabilities through [private vulnerability reporting](https://github.com/pooyanmajd/pmtechdev-mcp-plugins/security/advisories/new), following [SECURITY.md](SECURITY.md). Never put credentials, connected content, private identifiers, or vulnerability details in a public issue or discussion.

This is a best-effort open-source project. Support requests do not create a service-level agreement.
